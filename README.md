# Self-Healing AI Test Framework

![CI](https://github.com/AndreiTita99/Self-Healing-AI-Test/actions/workflows/ci.yml/badge.svg)

> A Playwright-based E2E test framework that generates tests from plain-English user stories
> and repairs broken element locators automatically at runtime — keeping normal runs fully
> deterministic and free.

---

## The problem

When a developer renames a CSS class or restructures the DOM, dozens of tests break even
though the app still works. Maintaining locators by hand is the single biggest maintenance
burden in test automation. This framework attacks that directly: tests never hardcode
selectors. When a locator breaks at runtime, an LLM inspects the live page, proposes a
validated replacement, and retries the action — turning a red build into a self-documenting
heal event that a human reviews before the registry is updated.

---

## Architecture

```
+-------------------+        +------------------------+
|  Test Generator   |        |   pytest test suite    |
|  (English -> JSON  | -----> |   (page objects +      |
|   -> test file)   |        |    locator registry)   |
+-------------------+        +-----------+------------+
                                         |
                              locator lookup fails (TimeoutError)
                                         v
                             +-----------------------+
                             |   Healing Engine      |
                             |  1. capture DOM       |
                             |  2. ask LLM for fix   |
                             |  3. VALIDATE candidate|
                             |  4. retry action      |
                             +-----------+-----------+
                                         |
                                         v
                             +-----------------------+
                             |  report.md / .json    |
                             |  old selector         |
                             |  proposed selector    |
                             |  confidence + reason  |
                             |  applied: true/false  |
                             +-----------------------+
```

## Quick start

```bash
# 1. Clone
git clone https://github.com/AndreiTita99/Self-Healing-AI-Test.git
cd Self-Healing-AI-Test

# 2. Create a virtual environment
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt
playwright install chromium

# 4. Configure environment (API key only needed for live healing + generation)
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY=sk-ant-...

# 5. Run the suite
pytest tests/ -v
```

The suite runs without an API key. The stub heal demo and all login tests pass out of the box.

## Usage

### Running the test suite

```bash
pytest tests/ -v                   # all tests, headless
pytest tests/ -v --headed          # show the browser
pytest tests/test_heal_demo.py -v  # healing demo only
LIVE_TESTS=1 pytest tests/ -v      # include live LLM healing test
```

### Generating tests from a story

```bash
# No API key needed — uses a pre-recorded plan:
python -m src.generator.generate \
  --plan stories/login.plan.json \
  --out  tests/test_generated_login.py

# Calls the Anthropic API (requires credits):
python -m src.generator.generate \
  --story stories/login.txt \
  --out   tests/test_generated_login.py
```

Generated files carry a `# Generated by …` header. Review them, optionally edit, then commit.

## Project structure

```
src/
  config.py              # typed settings via pydantic-settings (.env)
  llm/
    client.py            # thin Anthropic SDK wrapper
    prompts.py           # versioned prompt templates for healing and generation
  registry/
    locators.yaml        # logical_name -> selector + description + role + expected_text
    registry.py          # load / lookup — tests never hardcode selectors
  framework/
    page.py              # BasePage + HealableLocator
    healing.py           # HealingEngine: DOM capture -> LLM -> validate -> retry
    reporter.py          # collects HealEvents -> report.md + report.json
  generator/
    schema.py            # Pydantic models: Step, TestCase, TestPlan
    codegen.py           # deterministic code generator (no LLM)
    generate.py          # CLI: --story (LLM) or --plan (fixture) -> pytest file
stories/
  login.txt              # plain-English acceptance criteria
  login.plan.json        # pre-recorded plan fixture
tests/
  conftest.py            # fixtures and reporter hooks
  test_login.py          # hand-written login flow tests
  test_heal_demo.py      # self-healing demo (stub + live variants)
  test_generated_login.py  # generated from login.plan.json
.github/workflows/ci.yml
```

## Deliberately out of scope

- **Healing anything other than element locators** — assertion repair via AI would undermine
  the suite's ability to catch real regressions.
- **A web dashboard** — a clean terminal output and a markdown report are sufficient and
  easier to diff in code review.
- **Multiple LLM providers** — one provider behind one interface.
- **Visual regression, mobile, cross-browser matrices** — out of scope for a locator-healing
  framework.
